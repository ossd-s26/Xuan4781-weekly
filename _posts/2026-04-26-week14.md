### Open Source Licenses

This week we covered software licenses, which is something I always assumed was just fine print nobody reads.

The thing that clicked for me was the concept of **copyleft**. If you use GPL code in your project, your project also has to be open source. That's a completely different philosophy from licenses like MIT. Neither is wrong, but they push projects in very different directions.

<!--more-->

One thing I didn't know going in was how **Creative Commons licenses** fit into all of this. I always assumed CC and open source were basically the same thing, but CC licenses are meant for creative works, not software — so you'd use them for documentation or wikis, not your actual codebase.

A useful framing from the lecture was thinking about license choice in terms of your actual goal. If you want big companies to use your library with no friction, MIT makes sense. If you want to make sure nobody can make a closed-source knockoff of your tool, GPLv3 is the move. And if you're building a SaaS business and don't want AWS to just host your software without giving anything back, AGPL covers that.

### The AI Code Laundering Problem

One thing that bothered me was the AI code laundering problem. This is where you hand them an open source dependency and their AI reads only the public docs and rewrite a functionally identical replacement from scratch. Because the AI basically never looked at the source code, they claim it's legally clean from any license obligations.

It just feels wrong. Calling it legally clean doesn't make it ethically clean.
